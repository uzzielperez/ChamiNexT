## Slide 2 — The problem (engineers)

### Interview prep is broken for the post-AI job market

| Pain | Reality today |
|------|----------------|
| **Wrong skill** | LeetCode optimizes recall; modern roles require reasoning, communication, and shipping under AI-assisted workflows |
| **No portable proof** | A green check on HackerRank does not prove you can deploy a feature in 24–72 hours |
| **Expensive mocks** | Human mock interviews cost €50–150 each; ChatGPT alone has no rubric, no employer-facing profile |
| **Fragmented loop** | Practice, portfolio, and employer signal live in different tools |

**Who feels this most:** Junior–mid software engineers, career switchers, and AI/ML engineers targeting **€80k–€150k+** roles (US: $100k–$500k).

---

## Slide 3 — The problem (companies)

### Hiring still predicts performance poorly

| Pain | Cost |
|------|------|
| **Whiteboard theater** | Hours of senior eng time per loop; high false-positive / false-negative rates |
| **Static assessments** | Puzzle banks do not measure product thinking, decomposition, or ship velocity |
| **No standardized “ship URL” credential** | Recruiters cannot compare candidates on *what they built* under time pressure |
| **AI policy confusion** | Teams lack a framework for “AI-assisted attempt” vs cheating |

**Who feels this most:** Startups and scale-ups hiring **1–20 engineers per quarter**—too small for HackerRank Enterprise, too serious for ad-hoc take-homes.

---

## Slide 4 — Our thesis

> **Shipping beats solving. Reasoning beats memorization. Real output is the only signal that matters.**

ChamiNext is **not** a LeetCode clone. We are a **daily practice + evaluation platform** where:

1. Engineers run **adaptive AI interviews** (Socratic, rubric-scored).
2. Engineers complete **Ship Tests**—24h / 72h / 7-day builds with PM briefs and AI review.
3. Employers use **Interview Studio** to assign challenges, score thinking + shipping, and rank shortlists.

**One platform:** Practice → talent profile → hire.

4. **Practice → profile → Interview Studio** on one platform (not bolt-on hiring).

**Positioning line:** *LeetCode measures recall. We measure readiness.*

## Slide 17 — AWS & Amazon Bedrock: how you help us

### What we need from AWS (90-day pilot)

| Need | AWS service | How it helps |
|------|-------------|--------------|
| **Production LLM inference** | **Amazon Bedrock** (Claude / Llama) | Replace/supplement Groq for interview-agent + ship-test-evaluator with enterprise SLA, EU regions, cost controls |
| **Agent orchestration** | **Amazon Bedrock Agents** | Multi-step interviewer: rubric scoring → follow-up generation → session summary → profile update in one governed workflow |
| **Serverless API** | **AWS Lambda + API Gateway** | Migrate Netlify functions; scale interview spikes without cold-start pain at demo day |
| **Auth & data** | **Amazon Cognito + RDS/Aurora or Neon on AWS** | Server-side trials, subscriptions, usage counters (stop trial abuse) |
| **Rate limiting & cache** | **ElastiCache / MemoryDB** | Distributed rate limits per user/IP (production-readiness P0) |
| **Payments** | **Stripe on AWS** (or **Marketplace** later) | Interview Season one-time + B2B invoicing |
| **Observability** | **CloudWatch + X-Ray** | LLM latency, 429s, cost per interview |
| **Credits** | **AWS Activate / POC credits** | Fund 50k–200k interview sessions during pilot without burning runway |

### Bedrock Agents — concrete test plan

**Agent 1: Interview Conductor**  
- Tools: `get_problem`, `score_turn`, `next_followup`, `save_session`  
- Test: 100 simulated sessions across 3 tracks; measure rubric consistency (target: <10% score variance on replay).

**Agent 2: Ship Test Evaluator**  
- Tools: `fetch_repo`, `run_rubric`, `generate_feedback`, `update_profile`  
- Test: 50 submissions against human-labeled gold set (target: ≥0.75 correlation on shipping score).

**Agent 3: Employer Shortlist Assistant** (B2B demo)  
- Tools: `list_candidates`, `compare_scores`, `draft_summary`  
- Test: 5 design-partner hiring managers run live shortlist session.

### AWS GTM support (first paying customers)

1. **AWS Startups / Activate** — credits to run pilot at scale.  
2. **Co-sell narrative** — “Assess engineering talent on Bedrock-governed AI interviews + ship credentials.”  
3. **Intro to SMB customers** — startups on AWS hiring first engineers (natural ICP for Interview Studio).  
4. **Well-Architected review** — credibility for B2B security questionnaires.  
5. **Case study** — “ChamiNext assesses 10k AI-era interviews on Bedrock” after 90-day pilot.
