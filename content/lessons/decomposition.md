# Problem decomposition

Before you write code, clarify the problem.

1. **Restate the goal** in one sentence.
2. **List constraints** — time, memory, input size, edge cases.
3. **Break into subproblems** — each should be testable alone.
4. **Pick an approach** — brute force first, then optimize with a named pattern.

In interviews, saying your decomposition out loud is half the signal. Coach will drill this in mocks and Ship Tests too.
